# 존재하지 않는 공간 — 노드형 제작 스튜디오 명세

## 1. 목적

반복 제작할 수 있는 가사 없는 콘셉트 플레이리스트를 위해, 키워드에서 음악·이미지·구성·배포 계획을 파생하는 로컬 우선(node-based) 기획 도구를 만든다.

첫 프리셋은 다음 두 가지다.

1. **지배인 동지, 랭면이 불었습니다.** — 류경호텔 105층에서 퇴근하지 못한 지배인과 끝내 나타나지 않는 남자의 평양 여름 누아르.
2. **BEFORE THE NEXT FALL** — `Cyberpunk: Edgerunners 2`의 새 크루가 아직 비극이 되기 전을 반복하는 비공식 팬 플레이리스트. 공식 줄거리 예측이나 공식 사운드트랙 재현이 아니라, 1화 이전의 가능성을 독창적인 음악·이미지로 만든다.

이 도구는 완성 음악·이미지를 직접 생성하거나 Mureka API 키를 보관하지 않는다. 사용자가 Mureka와 이미지 생성기에 붙여 넣을 수 있는 프롬프트·제작 계획과, Mureka Melody ideas에 넣을 수 있는 30초 자체 모티프 MIDI를 만든다.

## 2. 사용자 흐름

1. 프리셋을 고른다.
2. 상단 FLOW에서 `1 콘셉트 → 2 키워드 → 3 사운드+구조 → 4 프롬프트 → 5 입력` 순서로 이동한다.
3. 현재 단계에 필요한 노드만 보고 핵심 변수와 키워드를 수정한다.
4. 3단계에서 자체 모티프 MIDI를 내려받고 트랙 수와 구조를 확정하면 4단계에 동일한 수의 트랙별 음악 프롬프트가 생긴다.
5. 메인 음악 프롬프트를 모든 트랙의 일관성 앵커로 쓰고, 각 트랙 프롬프트에는 해당 구간의 구조만 덧붙인다.
6. 모든 메인·트랙 프롬프트의 처음과 끝에 편집 불가능한 공통 무가사 잠금을 넣고 Mureka Instrumental 모드만 지시한다.
7. Mureka 음악 생성 Style 제한에 맞춰 `마스터 + 현재 트랙 구조` 최종 복사본을 1,000자 이내로 강제하고 실제 글자 수를 표시한다.
8. 이미지 프롬프트는 동일한 키워드·정서·색을 공유하며 음악 프롬프트와 연결해 표시한다.
9. 키워드에서 찾은 참고 이미지 10개를 원출처 링크와 함께 살펴본다.
10. 5단계에서 유튜브 영상 제목·설명·챕터·플레이리스트·해시태그·태그·고정 댓글과 Studio 설정값을 각각 복사한다.
11. `전체보기`에서 1–5단계의 핵심 결과와 전체 복사 패키지를 한 화면으로 검토한다.
11. 결과를 복사하거나 전체 프로젝트를 JSON/Markdown으로 내보낸다.
12. 값은 브라우저 `localStorage`에 프리셋별로 저장하고 JSON으로 다시 불러올 수 있다.

## 3. 기능 요구사항

- 한 화면에 모든 노드를 뿌리지 않고 다섯 개의 단계 화면으로 분리한다.
- 별도의 `전체보기`는 편집용 노드를 복제하지 않고 현재 프로젝트의 1–5단계 결과를 읽기 전용 요약으로 조합한다.
- 단계 안의 표준 노드는 포인터/터치로 이동할 수 있다.
- 노드 위치와 편집값을 단계·프리셋별로 자동 저장한다.
- 노드 잠금/해제와 단계별 배치 초기화를 지원한다.
- 프리셋마다 독립된 상태를 유지한다.
- 노드 연결선은 이동 후에도 다시 계산한다.
- 핵심 입력값: 콘셉트, 감정, BPM, 조성, 대표 길이, 금지 요소, 키워드 묶음.
- 파생 결과: 추천 메인 Mureka 프롬프트, 구조 수와 일치하는 트랙 프롬프트, 30초 자체 모티프 MIDI, 추천 이미지 프롬프트, 네거티브 프롬프트, 작품 제목, 챕터형 기승전결, 참고 이미지 10개, 프리셋 B의 비인가 방송 영상 문구와 팬 작품 고지, 유튜브 업로드·플레이리스트 입력 패키지.
- 모든 Zerune 음악은 무가사다. 엔진 수준의 공통 잠금으로 가사·노래·대사·허밍·코러스·랩·보컬 찹·음절·비언어 발성을 금지하고 Mureka Instrumental 모드만 지시한다.
- 텍스트 결과는 개별 복사할 수 있다.
- 전체 결과는 JSON과 Markdown으로 내보낼 수 있다.
- 외부 요청, 추적, API 키 입력란이 없는 정적 앱이어야 한다.
- 배포 캘린더와 3단계 검수는 화면 노드에서 제외하고 문서의 고정 체크리스트로 유지한다.
- 참고 이미지는 그대로 재배포하는 자산이 아니라 장면·렌즈·노출·시간대를 연구하는 미리보기이며 원출처로 이동할 수 있어야 한다.

## 4. 권장 길이와 검증 가설

길이는 플랫폼의 공식 규칙이 아니라 표본과 사용 맥락에서 도출한 시작 가설이다.

| 역할 | 길이 | 목적 |
|---|---:|---|
| 초기 공개 | 30분 | 첫 3–5편의 제작비를 낮추고 제목·썸네일·첫 30초 반응 확인 |
| 길이 비교 | 60분 | 30분 작품의 유지율이 확인된 뒤 같은 콘셉트로 비교 |
| 장시간편 | 120분 이상 | 채널과 사용 목적이 확립된 뒤 집중·게임·수면용으로 별도 실험 |

프리셋 A의 초기 공개편 기본 전개:

- 0:00–1:39: 문턱/콜드 오픈 — 7–12초 안에 영상의 정체성을 보이고, 20초 안에 핵심 음색을 제시한다.
- 1:39–6:34: 공간 확립.
- 6:34–14:04: 탐색과 의식.
- 14:04–19:41: 금지된 사실/추격의 긴장.
- 19:41–25:19: 감정적 절정.
- 25:19–30:00: 잔향과 루프 가능한 귀환.

일반 프리셋은 트랙/챕터 9–12개를 가설로 둔다. 프리셋 A는 명시적 5막을 보존하기 위해 30분·5개 트랙을 사용한다. 프리셋 B는 사용자가 정한 55–65분 범위의 중앙값인 60분과 10개 트랙을 쓴다. 10개는 공식 사이트가 밝힌 10부작과 구조적으로만 대응하며, 실제 에피소드 내용·순서·결말을 예측하지 않는다. 전환 크로스페이드는 6–12초를 시작값으로 쓰며 갑작스러운 무음이나 축제형 EDM 드롭은 기본적으로 피한다.

검증 지표는 조회수 단독이 아니라 노출 클릭률, 첫 30초 유지, 평균 시청시간, 핵심 구간 유지율, 재방문 시청자로 둔다. 비슷한 길이의 최근 영상끼리 비교한다.

## 5. 데이터 모델

```text
preset
├── meta: id, name, eyebrow, status
├── theme: subject, setting, ritual, emotion, audiencePromise
├── music: bpm, key, meter, genres, instruments, texture, exclusions
├── referenceAudio: mode, 30s duration, bpm, MIDI notes, recipe, rights
├── visual: subject, setting, camera, palette, texture, narrative, exclusions
├── keywords: location, ritual, sensory, emotion, music, discovery
├── duration: pilot, flagship, extended, chapters, crossfade
├── story: six phases with minute ranges and energy values
├── campaign: relative timing, deliverable, goal
├── rights: originality and disclosure rules
├── video: intro, interludes, ending, loop, disclaimer
├── youtube: descriptionLead, playlistTitle, playlistDescription, hashtags, tags, pinnedComment, disclosure
└── nodes: id, kind, title, x, y, inputs, outputs
```

저장 스키마:

```text
zerune-studio/v6/<preset-id>
├── version
├── updatedAt
├── fields
├── positions
├── lockedNodes
└── customNodes
```

## 6. 노드 그래프

```text
FLOW 1 · 콘셉트
[작품 제목 · 고정] ─> [장면과 핵심 정서] ─> [화자 · 관계 · 미해결]

FLOW 2 · 키워드
[장소·의식·감각·정서] ─> [음악 키워드] + [이미지 키워드]

FLOW 3 · 사운드 + 구조
[사운드 설계] ─> [Mureka 30초 자체 소스] ─> [목표 길이 N개 트랙 구조]

FLOW 4 · 프롬프트
[추천 메인 음악 프롬프트]
          ├─ + [트랙 01 구조] ─> [트랙 프롬프트 01]
          ├─ + [트랙 02 구조] ─> [트랙 프롬프트 02]
          └─ + [트랙 0N 구조] ─> [트랙 프롬프트 0N]
          └─ 동일 장면·정서·색 ─> [추천 이미지 프롬프트 + 방송 문구] ─> [참고 이미지 10개]

FLOW 5 · 입력
[유튜브 영상 제목 + 설명·챕터] ─> [플레이리스트 + 검색·댓글] ─> [Studio 설정값 + 업로드 순서]

전체보기
[01 콘셉트] + [02 키워드] + [03 사운드·구조] + [04 프롬프트] + [05 유튜브 입력]
                                      └─> [전체 복사 패키지]
```

화면에는 현재 FLOW의 노드만 보인다. 연결선은 해당 단계 안의 정보 흐름만 간결하게 안내한다.

## 7. YouTube 입력 패키지

모든 프리셋은 같은 5단계 구조를 사용하되 문구는 프리셋 데이터에서 파생한다.

- 영상 제목: 추천 작품 제목을 사용하고 현재 글자 수를 표시한다. YouTube 제한인 100자를 넘으면 경고한다.
- 영상 설명: 첫 문단에 작품의 핵심 약속을 두고, 수동 챕터·작품 설명·권리/팬 고지·해시태그를 한 번에 붙여 넣을 수 있게 합친다. 5,000자 제한을 표시한다.
- 챕터: `00:00`으로 시작하고 오름차순 3개 이상, 각 10초 이상이라는 공식 요건을 자동 충족한다.
- 플레이리스트: 이름과 설명을 별도 복사 블록으로 제공한다. 공개 범위는 사용자가 YouTube에서 최종 선택한다.
- 발견 정보: 해시태그는 핵심 3개만 사용하고, 태그는 오탈자·한영 표기 보조용으로 제한한다. 태그를 설명란에 나열하지 않는다.
- 고정 댓글: 작품의 한 문장 질문과 권리 고지가 필요한 경우 이를 함께 제공한다.
- Studio 설정: 카테고리 `음악`, 아동용 아님, 원본 언어, 수동 챕터, 합성 콘텐츠 고지, 유료 프로모션 여부, 녹화 장소, 라이선스·공개 상태의 확인값을 보여준다.
- 합성 콘텐츠: Mureka 등으로 합성 음악을 만들면 YouTube의 `변경되거나 합성된 콘텐츠`를 `예`로 선택한다. 현실적으로 보이는 합성 장소·사건도 같은 원칙을 쓴다.
- 썸네일: 16:9, 권장 3840×2160, 최소 너비 640px을 체크리스트에 표시한다. 썸네일 문구가 없는 프리셋은 이를 그대로 보존한다.
- 앱은 YouTube 계정에 로그인하거나 업로드하지 않으며, 복사 가능한 입력값만 만든다.

## 8. 프리셋 A 방향

- 제목: `지배인 동지, 랭면이 불었습니다.`만 사용한다. 마침표를 포함하고 부제·장르·플레이리스트 문구를 붙이지 않는다.
- 한 문장: 류경호텔 105층에서 퇴근하지 못한 지배인이, 끝내 오지 않은 남자의 목소리를 듣는 평양의 여름 로맨스.
- 관계: 남자는 기다리는 사람·과거의 사람·이미 떠난 사람·남겨진 기억·지배인만 듣는 목소리 중 무엇으로도 해석될 수 있으며 정답을 밝히지 않는다.
- 구조: `시작 → 기다림 → 목소리 → 절정 → 마지막` 5막을 기본 트랙 5개와 정확히 연결한다.
- 길이: 초기 3–5편은 30분으로 고정하고 `3:54 → 11:24 → 18:00 → 25:12 → 30:00` 경계를 사용한다.
- 음악: 평양 누아르 로맨스 앰비언트. 앰비언트·전자음향 70%, 북한 개량 민족악기에서 착안한 절제된 질감 30%를 사용한다.
- 소스: 텍스트만으로 먼저 생성하고, 모티프 일관성이 필요할 때만 자체 30초 MIDI를 Melody ideas에 넣는다. 외부 상업곡은 레퍼런스로 업로드하지 않는다.
- 이미지: 무더운 여름밤, 랭면 두 그릇, 빈 두 자리, 의자에 걸린 지배인 재킷, 호텔 열쇠, 녹은 얼음, 반쯤 열린 문, 결로 낀 통유리를 사용한다.
- 금지: 실제 대사·보컬·기존 북한 노래·혁명가극·아리랑·흥겨운 국악 퓨전·과장된 신파·선전음악·사람의 얼굴이나 몸·썸네일 문구.
- 원칙: 관계와 결말을 설명하지 않고 흔적만 남긴다. 감정의 이름을 직접 말하지 않는다.

## 9. 프리셋 B 방향

- 제목: `BEFORE THE NEXT FALL — Cyberpunk: Edgerunners 2 Fan Playlist`. 한글 병기는 `다음 추락이 오기 전 — 사이버펑크: 엣지러너 2 팬 플레이리스트`를 별도 후보로 둔다.
- 정의: 새 크루가 아직 나이트 시티의 비극이 되기 전, 선택이 일어나지 않은 시간을 반복하는 비공식 팬 플레이리스트. 핵심 정서는 기대감보다 `아직 일어나지 않은 상실에 대한 그리움`이다.
- 제목 의미: `Fall 2026`, 엣지러너의 추락과 몰락, 엣지에서 떨어지는 운명의 세 의미를 함께 쓴다.
- 구조: 60분·10곡. `NEW CREW DETECTED`에서 접속하고 `THE CITY KEEPS THE ENDING`에서 인간적 4음 모티프가 사라진 뒤 첫 곡의 부팅음으로 돌아간다. 공식 에피소드의 줄거리나 결말은 예측하지 않는다.
- 음악: cybernetic dark ambient + industrial electronica + restrained breakbeat. 96–106 BPM, 체감 48–53 BPM 하프타임, FM bass, 금속 타격, 유압음, 전파 노이즈, 필름 릴·셔터·손상 방송 신호를 사용한다.
- 일관성: 모든 곡에 같은 3–4음 인간적 모티프를 둔다. 9곡에서 잠시 멈추고 10곡에서 사라지며 첫 곡에서 재등장한다.
- 이미지: 본편 스틸 모사가 아니라 `1화 이전에 유출된 비인가 촬영 기록`. 공식 캐릭터와 닮지 않은 익명 원본 크루, 아날로그 카메라, CCTV 관제벽, 필름 릴, 젖은 도시를 사용한다.
- 영상 문구: `UNAUTHORIZED BROADCAST`, `RECORDED BEFORE EPISODE ONE`, `CREW STATUS: ALIVE`, `ENDING: NOT YET RECORDED`; 중간과 마지막 문구도 복사 가능한 결과로 제공한다.
- 금지: 기존 캐릭터·의상·로고·공식 스틸·공식 사운드트랙 선율·대사·특정 스튜디오 화풍 직접 모사·실제 줄거리처럼 보이는 예측.
- 일정: 공식 사이트가 밝힌 `Fall 2026`만 사실로 표시한다. 정확한 날짜는 미정이며 상대 일정도 이번 화면 노드에서는 다루지 않는다.

## 10. 보안·권리 경계

- 정적 브라우저 앱에 Mureka API 키를 넣지 않는다.
- Mureka 파일 입력에는 자체 MIDI·직접 녹음·직접 합성한 소리만 사용한다.
- 생성물의 상업 이용 전에는 사용 시점의 Mureka 약관과 각 생성 도구의 라이선스를 다시 확인한다.
- 프랜차이즈 이름은 제목·검색·비평 문맥에서 팬 작품을 식별하는 데만 제한적으로 사용하고, 생성 프롬프트는 독창적이고 일반적인 시각 언어로 작성한다.
- 프리셋 B의 설명과 내보내기에는 `This is an unofficial fan work and is not approved or endorsed by CD PROJEKT RED.`를 분명히 넣는다.
- 공식 음악은 제3자 권리가 얽힐 수 있으므로 오디오 소스로 업로드하거나 생성 프롬프트에서 재현하지 않는다.
- 실제 정치 인물, 국기, 선전물은 프리셋 기본값에서 제외한다.

## 11. 완료 조건

- 두 프리셋, 다섯 단계와 전체보기가 전환되고 상태가 서로 섞이지 않는다.

- 3단계의 트랙 수와 5단계 설명의 수동 챕터 수가 항상 일치한다.
