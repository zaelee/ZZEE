# 3단계 검토 기록

최종 재검토일: 2026-07-15 (Asia/Seoul)

## PASS 1 — 콘텐츠 / 흐름

상태: **통과**

- 한 화면에 많은 노드를 뿌리던 구조를 `1 콘셉트 → 2 키워드 → 3 사운드+구조 → 4 프롬프트`로 분리했다.
- 배포 캘린더·검수·리서치 노드는 화면에서 제거하고 문서의 고정 정보로 남겼다.
- 각 단계에는 관련 노드만 2–3개 표시되고, 상단 FLOW에서 완료한 이전 단계를 확인할 수 있다.
- 평양 프리셋 제목을 류경호텔 고유명사보다 `평양·고위층·랭면`이 먼저 읽히는 문장으로 교체했다.
- 추천 제목은 `평양 고위층도 랭면엔 식초를 칠까? | 105층의 금지된 저녁`이며 추천 배지와 복사 버튼으로 강조했다.
- 썸네일 후편집 문구는 `고위층도 식초 칠까?`로 맞췄다.

## PASS 2 — 구조 / 생성 일관성

상태: **통과**

- 3단계의 트랙 수와 4단계의 트랙 프롬프트 수가 동일한 생성 함수에서 만들어져 항상 일치한다.
- 기본값은 류경호텔 10개, 사이버펑크 11개 트랙이다.
- 트랙 수를 변경해도 기본 서사 전체를 균등하게 재표본화해 첫 문턱과 마지막 귀환이 유지된다. 브라우저에서 10개를 7개로 바꿨을 때 구조 7개·프롬프트 7개·마지막 `귀환`을 확인했다.
- 추천 메인 음악 프롬프트에 장르·BPM·조성·악기·질감·반복 모티프·제외 조건을 고정했다.
- 모든 트랙 프롬프트는 메인 프롬프트 전문으로 시작하고 시간·에너지·전경 악기·서사 기능만 덧붙인다.
- 이미지 프롬프트는 음악의 BPM·감정·공간·색을 공유하며 음악 노드에서 이미지 노드로 연결된다.
- 추천 이미지 프롬프트와 렌즈·노출·시간대 변주를 분리해 표시한다.

## PASS 3 — 참고 이미지 / 기능 / 보안

상태: **통과**

- 두 프리셋에 키워드별 참고 프레임 10개씩을 추가했다.
- 류경호텔 프리셋은 외관·평양 야경·랭면 정물·호텔 식당을 섞고, 사이버펑크 프리셋은 실제 빗길·네온 거리·도로·브루탈리즘 건축을 섞었다.
- 모든 카드에 검색 키워드, 렌즈·시점·노출·시간대 변주 메모, 원본·라이선스 링크, 저작자 표기를 제공한다.
- 실제 브라우저에서 두 프리셋의 10개 이미지가 모두 로딩되는 것을 `naturalWidth > 0`으로 확인했다.
- 트랙 프롬프트는 기본적으로 접혀 있고 필요한 트랙만 펼쳐 전문을 복사할 수 있다.
- 단계 전환, 프리셋 분리, 결과 갱신, 추천 강조, JSON/Markdown 내보내기 코드를 유지했다.
- 외부 이미지는 표시만 하며 앱에서 다운로드·재배포하지 않는다. JSON 가져오기는 허용된 필드와 노드만 저장한다.
- API 키, 쿠키, 인증, 외부 추적, Mureka 직접 호출이 없다.
- 사용자 문자열은 `innerHTML`이 아니라 DOM `textContent`로 렌더링한다.
