# 3단계 검토 기록

최종 재검토일: 2026-07-16 (Asia/Seoul)

## PASS 1 — 콘텐츠 / 흐름

상태: **통과**

- 한 화면에 많은 노드를 뿌리던 구조를 `1 콘셉트 → 2 키워드 → 3 사운드+구조 → 4 프롬프트 → 5 입력`으로 분리했다.
- 배포 캘린더·검수·리서치 노드는 화면에서 제거하고 문서의 고정 정보로 남겼다.
- 각 단계에는 관련 노드만 2–3개 표시되고, 상단 FLOW에서 완료한 이전 단계를 확인할 수 있다.
- 평양 프리셋 제목은 `지배인 동지, 랭면이 불었습니다.` 한 문장만 남겼고 마침표까지 제목으로 고정했다.
- 부제·장르명·플레이리스트·믹스·시간 표기를 제목 앞뒤에 붙이지 않으며, Markdown의 첫 줄도 이 문장만 사용한다.
- 콘셉트 FLOW를 `작품 제목 → 장면과 핵심 정서 → 화자·관계·미해결` 세 노드로 재구성했다.
- 사용자가 확정한 네 문단 소개 카피를 `작품 소개문 · 고정 카피` 필드와 Markdown 내보내기에 그대로 반영했다.
- 남자의 정체와 두 사람의 관계, 오지 못한 이유, 지배인이 떠났는지 여부를 열어 두었다.
- 썸네일 문구를 제거하고 두 그릇·빈 두 자리·재킷·호텔 열쇠·반쯤 열린 문이 제목의 의미를 대신하게 했다.
- 두 번째 프리셋을 기존 배달원 콘셉트에서 `BEFORE THE NEXT FALL`로 전면 교체했다. 추천 제목, 한글 병기, Inspired 대안과 Fall의 세 의미를 작품 제목 노드에서 함께 확인할 수 있다.
- 두 번째 작품은 `Everyone is still alive before episode one`을 핵심 원칙으로 두고, 공개된 공식 설정까지만 정서적 출발점으로 사용한다. 6곡 이후의 개념은 공식 줄거리 예측이 아닌 팬 창작 해석으로 분리했다.
- 썸네일 카피와 `UNAUTHORIZED BROADCAST` 오프닝, 중간 신호, 마지막 `SEE YOU IN FALL`, 01번 부팅음 루프를 이미지 결과 노드에 추가했다.
- 모든 프로젝트에 `05 입력`을 추가해 영상 제목·통합 설명·수동 챕터·재생목록·해시태그·태그·고정 댓글·썸네일 사양·Studio 설정·업로드 순서를 입력칸별 복사 블록으로 제공한다.
- 별도 `전체보기`에서 01–05의 핵심 결과와 트랙별 프롬프트를 접힌 상태로 확인하고 전체 프로젝트 텍스트를 한 번에 복사할 수 있다.

## PASS 2 — 구조 / 생성 일관성

상태: **통과**

- 3단계의 트랙 수와 4단계의 트랙 프롬프트 수가 동일한 생성 함수에서 만들어져 항상 일치한다.
- 평양 여름 프리셋은 `시작 → 기다림 → 목소리 → 절정 → 마지막` 5막과 음악 프롬프트 5개를 정확히 연결한다.
- 초기 3–5편 기본 길이를 30분으로 고정했다. 류경호텔의 13%·25%·22%·24%·16% 가중치는 `0:00–3:54–11:24–18:00–25:12–30:00`으로 이어진다.
- 트랙 수를 변경해도 첫 `시작`과 마지막 `마지막`을 포함한 전체 서사를 다시 표본화한다.
- 추천 메인 음악 프롬프트에 무가사 잠금·장르·BPM·조성/비율 또는 장면/서사 규칙·제외 조건을 우선순위로 압축하고, Mureka에 전체 30분을 한 번에 생성하라는 문장은 제거했다.
- Mureka Style 입력 제한에 맞춰 마스터와 트랙 구조의 합본을 1,000자 이내로 강제하고, 긴 사용자 입력에서도 제한을 넘지 않는 자동 테스트와 화면 글자 수 표시를 추가했다.
- 앰비언트·전자음향 70%와 개량 민족악기에서 착안한 질감 30%를 명시하고, 실제 대사·보컬·기존 북한 노래·혁명가극·아리랑·과장된 신파·선전풍 합주를 제외했다.
- 모든 트랙 프롬프트는 메인 프롬프트 전문으로 시작하고 현재 막·에너지·전경 음색/서사 기능·크로스페이드 여백만 덧붙인다. 한 번에 현재 막 하나만 생성하도록 명시했다.
- FLOW 3에 `Mureka 30초 자체 소스` 노드를 추가했다. 프리셋별 자체 4음 모티프 MIDI를 브라우저에서 내려받을 수 있고, 텍스트 우선 → Melody ideas 보조 → 필요할 때만 자체 Reference 오디오 순서를 안내한다.
- 이미지 프롬프트는 음악의 BPM·감정·공간·색을 공유하며 음악 노드에서 이미지 노드로 연결된다.
- 추천 이미지 프롬프트와 렌즈·노출·시간대 변주를 분리해 표시한다.
- 두 번째 프리셋은 60분·10곡을 `0:00–4:48–10:12–16:12–22:12–28:12–34:48–41:24–48:36–54:36–60:00`으로 정확히 나눈다. 영문·한글 트랙명을 사운드 구조와 Mureka 결과에서 같은 순서로 표시한다.
- 96–106 BPM 범위 안의 102 BPM, 체감 51 BPM 하프타임, cybernetic dark ambient + industrial electronica + restrained breakbeat, 모든 곡의 4음 인간 모티프를 고정했다.
- 10곡째에서 인간 모티프를 없애고 1곡째 부팅음으로 돌아오게 했다. Mureka에는 한 번에 한 곡만 요청하고 필요한 길이는 Extension과 편집으로 맞춘다.
- 두 번째 Mureka 생성 프롬프트에는 프랜차이즈 이름을 넣지 않는다. 공식 OST·대사·캐릭터 음성 대신 자체 30초 MIDI와 직접 합성한 FM bass·셔터·금속·방송 노이즈만 소스로 허용한다.
- YouTube 패키지는 구조 노드와 같은 트랙 계획에서 챕터를 생성한다. 제목 100자·설명 5,000자·`00:00` 시작·3개 이상·구간당 10초 이상 조건을 자동 점검한다.
- Mureka 음악과 생성형 이미지를 쓰는 두 프로젝트 모두 `변경되거나 합성된 콘텐츠: 예`를 기본 Studio 설정으로 제공하고, 팬 프로젝트의 비공식 고지를 설명·재생목록·고정 댓글에 유지한다.

## PASS 3 — 참고 이미지 / 기능 / 보안

상태: **통과**

- 두 프리셋에 키워드별 참고 프레임 10개씩을 추가했다.
- 두 번째 프리셋의 앞쪽 참고 프레임을 CCTV 관제벽·CRT·필름 릴로, 뒤쪽을 영화 카메라·젖은 도시로 교체하고 모든 도시 프레임의 변주 메모를 `1화 이전 비인가 기록`에 맞췄다.
- 평양 프리셋의 기존 외관·야경·랭면·호텔 식당 원본은 유지하되 모든 참고 방향을 무더운 여름·결로·두 그릇·재킷·열쇠·반쯤 열린 문으로 다시 작성했다.
- 모든 카드에 검색 키워드, 렌즈·시점·노출·시간대 변주 메모, 원본·라이선스 링크, 저작자 표기를 제공한다.
- 실제 브라우저에서 두 프리셋의 10개 이미지가 모두 로딩되는 것을 `naturalWidth > 0`으로 확인했다.
- 트랙 프롬프트는 기본적으로 접혀 있고 필요한 트랙만 펼쳐 전문을 복사할 수 있다.
- 단계 전환, 프리셋 분리, 결과 갱신, 추천 강조, JSON/Markdown 내보내기 코드를 유지했다.
- v3–v5 저장값을 읽어 v6 구조로 옮기고, 두 프리셋의 노드 위치와 내용은 계속 분리 저장한다.
- 모바일에서 저장된 현재 단계가 05일 때 가로 FLOW가 자동으로 활성 탭까지 이동하도록 보완했다.
- 외부 이미지는 표시만 하며 앱에서 다운로드·재배포하지 않는다. JSON 가져오기는 허용된 필드와 노드만 저장한다.
- API 키, 쿠키, 인증, 외부 추적, Mureka 직접 호출이 없다.
- 상업음원·기존 북한 노래·애니메이션 OST·타인의 연주를 Mureka 입력으로 업로드하지 않는 권리 경계를 표시했다.
- CD PROJEKT RED 팬 콘텐츠 가이드라인의 비공식 고지 문구를 앱 상태·영상 계획·Markdown에 넣었고, 공식 음악의 제3자 권리 가능성을 리서치와 소스 금지 조건에 남겼다.
- 사용자 문자열은 `innerHTML`이 아니라 DOM `textContent`로 렌더링한다.

## 자동·시각 검사

- 저장소 전체 `npm run check` 통과.
- Zerune Studio 테스트 19개를 포함한 전체 24개 테스트 통과.
- JavaScript 문법 및 로컬 링크 검사 통과.
- 실제 브라우저 콘솔 오류·경고 0건.
- 데스크톱 1600×1000 및 모바일 390×844 확인.
- 데스크톱과 모바일 모두 05 입력 노드가 겹치거나 보드 아래로 잘리지 않음을 좌표로 확인했다. 두 번째 프로젝트의 가장 긴 Studio 노드 하단은 3376px, 보드 하단은 3600px다.
- Markdown 다운로드에 정확한 제목, 화자·관계, 5개 음악 프롬프트, 무문구 이미지 프롬프트가 포함되는 것을 확인했다.
- 두 번째 Markdown 다운로드에 10개 음악 프롬프트, 영상 방송 문구, 팬 작품 고지가 포함되는 것을 확인했다.
- 실제 브라우저에서 05 전체 패키지 복사 토스트와 Markdown 다운로드를 실행했고, 내려받은 파일에 YouTube 제목·통합 설명·`00:00` 챕터·합성 콘텐츠 설정이 포함되는 것을 확인했다.
- 전체보기는 첫 프로젝트 5개, 두 번째 프로젝트 10개의 접힌 트랙 프롬프트를 표시하며 두 프로젝트 모두 5개 섹션을 유지한다.
- 두 번째 프리셋의 MIDI 다운로드를 실제 브라우저에서 실행하고 `Standard MIDI data (format 0), 1 track, 1/480`으로 확인했다.
- 두 번째 프롬프트 화면에서 이미지 노드 하단과 참고 노드가 겹치는 문제를 발견해 참고 노드 기본 위치를 2280px로 내리고 보드를 3400px로 늘렸다. 최종 간격은 94px다.
